Create `src/extractor.py` with the following components:

**1. UrgencyTier enum** (using str enum for easy string comparison):
- CRITICAL = "Critical" (action date <= 7 days from now)
- HIGH = "High" (action date <= 30 days from now)
- NORMAL = "Normal" (action date > 30 days or no action date)

**2. MajorUpdateFields dataclass:**
- mc_id: Optional[str] — e.g. "MC1234567"
- action_required_date: Optional[datetime] — parsed deadline
- affected_services: list[str] — e.g. ["Exchange Online", "Teams"]
- categories: list[str] — e.g. ["MAJOR UPDATE", "ADMIN IMPACT"]
- published_date: datetime — from email.received_datetime
- last_updated_date: datetime — from email.received_datetime (same for single version; updated during dedup)
- body_preview: str — first 200 chars of body_content
- urgency: UrgencyTier — calculated from action_required_date
- is_updated: bool — True if a newer version of this MC ID was seen (set during dedup)
- subject: str — original email subject

**3. MessageCenterExtractor class** with compiled regex patterns:

Patterns (compile at class level for performance):
- MC_ID_PATTERN = re.compile(r'\bMC(\d{5,7})\b', re.IGNORECASE) — matches MC IDs with 5-7 digits
- ACTION_DATE_PATTERN = re.compile(r'(?:action required by|act by|deadline[:\s]|by)\s*(\w+ \d{1,2},?\s*\d{4}|\d{1,2}/\d{1,2}/\d{4})', re.IGNORECASE) — matches both "March 15, 2026" and "03/15/2026" formats
- SERVICE_PATTERN = re.compile(r'\b(Exchange Online|Microsoft 365 Apps|Microsoft Teams|Teams|SharePoint Online|SharePoint|OneDrive for Business|OneDrive|Power Platform|Power BI|Power Automate|Power Apps|Microsoft Viva|Viva|Microsoft Intune|Intune|Microsoft Entra|Entra|Windows|Outlook|Microsoft Purview|Purview|Microsoft Copilot|Copilot)\b', re.IGNORECASE)
- CATEGORY_PATTERN = re.compile(r'\b(MAJOR UPDATE|ADMIN IMPACT|USER IMPACT|RETIREMENT|BREAKING CHANGE|ACTION REQUIRED)\b', re.IGNORECASE)

Methods:
- `extract(self, email: Email) -> MajorUpdateFields` — Extract all fields from a single email. Searches subject first for MC ID, falls back to body. Parses dates with multiple format attempts (MM/DD/YYYY via strptime, also "Month DD, YYYY" via strptime with %B %d, %Y and %B %d %Y). Body preview: first 200 chars of body_content, truncated at word boundary with "...". Urgency calculated via `_calculate_urgency()`.
- `extract_batch(self, emails: list[Email]) -> list[MajorUpdateFields]` — Extract fields from multiple emails. Calls extract() for each.
- `deduplicate(self, fields_list: list[MajorUpdateFields]) -> list[MajorUpdateFields]` — Group by mc_id. For each MC ID with multiple entries, keep the one with latest published_date. Set is_updated=True on the kept entry if duplicates existed. Entries with mc_id=None are always kept (no dedup possible). Returns deduplicated list.
- `_calculate_urgency(self, action_date: Optional[datetime]) -> UrgencyTier` — No date returns NORMAL. Calculate days_remaining = (action_date.date() - datetime.now().date()).days. <=7 = CRITICAL, <=30 = HIGH, >30 = NORMAL. Past dates (negative days) = CRITICAL.
- `_extract_mc_id(self, email: Email) -> Optional[str]` — Search subject first, then body_content. Return "MC{digits}" format or None.
- `_extract_action_date(self, email: Email) -> Optional[datetime]` — Search body_content with ACTION_DATE_PATTERN. Try parsing with multiple strptime formats. Return None on failure, log warning.
- `_extract_services(self, email: Email) -> list[str]` — findall on body_content + subject, deduplicate (preserve order using dict.fromkeys), normalize capitalization (title case for display).
- `_extract_categories(self, email: Email) -> list[str]` — findall on body_content + subject, deduplicate, uppercase for display.
- `_make_body_preview(self, body_content: str, max_length: int = 200) -> str` — Truncate at word boundary, add "..." if truncated.

Follow project conventions:
- Module-level logger: logger = logging.getLogger(__name__)
- Relative imports: from .ews_client import Email
- Type hints throughout (Python 3.10+ style)
- Private methods with single underscore prefix
- snake_case for functions, PascalCase for classes

- `major_update_word_date` — Email with date in "March 15, 2026" format
