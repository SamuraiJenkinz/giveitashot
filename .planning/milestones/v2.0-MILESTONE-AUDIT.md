### Broken Flow Detail

**`--major-only` flag (src/main.py:286-288)**

`summarizer = EmailSummarizer()` is assigned inside `if not args.major_only:` (line 196), but `summarizer.format_major_updates_html()` is called unconditionally at line 288. When `--major-only` is active, the regular digest block is skipped, `summarizer` is never assigned, and line 288 raises `UnboundLocalError`.

This is a **pre-existing orchestration bug**, not a migration defect. The standard execution path (no `--major-only` flag) works correctly because `summarizer` is always created in the regular digest block before reaching line 288. The `--major-only` flag was introduced before v2.0 and this code path was not exercised by any existing test.
