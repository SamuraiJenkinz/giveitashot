  <action>
  1. Add `pydantic>=2.8.0` to requirements.txt (after httpx line). Run `pip install pydantic>=2.8.0` to install.

  2. Update `.env.example`: Change the API_VERSION comment and default from `2023-05-15` to `2024-08-01-preview`. Add a comment explaining this is required for structured outputs (AI action extraction). Keep the old value in a comment for reference.

  3. Create `src/action_extractor.py` with:

  **Pydantic Models:**
  - `AdminAction(BaseModel)`: Fields: `action: str` (Field, max_length=50, description="Short imperative: 'Update auth settings'"), `details: Optional[str]` (Field, default=None, max_length=200), `role: Optional[str]` (Field, default=None, description="Target admin role e.g. 'Global Admin'")
  - `ActionExtraction(BaseModel)`: Fields: `actions: list[AdminAction]` (Field, default_factory=list, max_length=5), `confidence: str` (Field, pattern="^(HIGH|MEDIUM|LOW)$")

  **ActionExtractor class:**
  - `__init__(self)`: Read Config.OPENAI_ENDPOINT and Config.OPENAI_API_KEY. Store as instance vars. If either is missing, log a warning and set `self._available = False` (do NOT raise — this allows graceful degradation when Azure OpenAI is not configured). Set `self._available = True` when both are present.
  - `@property available(self) -> bool`: Returns self._available.
  - `extract_actions(self, update: MajorUpdateFields) -> Optional[ActionExtraction]`: Extract actions from a single major update.
    - If not self._available, return None immediately.
    - Build system prompt instructing the LLM to extract 1-5 admin actions as short imperative commands (3-5 words). Include rules: prioritize most important, imperative verb form, include role if task is role-specific, set confidence HIGH if actions clearly stated, MEDIUM if partially stated, LOW if inferred.
    - Build user prompt with: MC ID, subject, action_required_date (formatted if present, "None" if not), body content truncated to first 1500 chars.
    - Build payload with messages, temperature=0.2, max_tokens=500, and response_format using json_schema type with ActionExtraction.model_json_schema() and strict=True.
    - Use httpx.Client(timeout=60.0) to POST to self._endpoint with headers (Content-Type + api-key).
    - Parse response: extract choices[0].message.content, validate with ActionExtraction.model_validate_json().
    - Catch httpx.HTTPStatusError, httpx.TimeoutException, and pydantic.ValidationError. On any failure, log warning with update.mc_id and error, return None.
  - `extract_actions_batch(self, updates: list[MajorUpdateFields]) -> dict[str, Optional[ActionExtraction]]`: Loop over updates, call extract_actions for each, store results keyed by mc_id (or a generated key like "unknown_N" for updates with mc_id=None). Log summary at end (X of Y succeeded).

  **Follow existing patterns from llm_summarizer.py:**
  - Same header construction (Content-Type + api-key)
  - Same httpx.Client usage with 60s timeout
  - Same error logging style
  - Import Config from .config, MajorUpdateFields from .extractor
  - Use stdlib logging (logger = logging.getLogger(__name__))

  **Do NOT:**
  - Create a custom exception class (use None return for failures, matching graceful degradation pattern)
  - Use the openai SDK (project uses direct REST via httpx)
  - Add retry logic (fail fast, send digest without actions)
