  <name>Task 1: Create EmailClassifier module and ClassificationResult dataclass</name>
  <files>src/classifier.py</files>
  <action>
Create `src/classifier.py` with the following implementation:

1. **ClassificationResult dataclass** with fields:
   - `is_major_update: bool` - whether email is classified as major update
   - `confidence_score: float` - weighted score (0-100)
   - `matched_signals: list[str]` - which signals matched (e.g., "sender_domain", "subject_mc_number", "body_keywords")

2. **EmailClassifier class** with:
   - Class-level COMPILED regex patterns (compile once, reuse):
     - `SENDER_PATTERN = re.compile(r'@email2\.microsoft\.com$', re.IGNORECASE)` for sender domain
     - `MC_NUMBER_PATTERN = re.compile(r'\bMC\d{5,7}\b')` for MC number in subject (5-7 digits to be slightly flexible)
     - `MAJOR_UPDATE_KEYWORDS = re.compile(r'\b(major update|retirement|admin impact|action required|breaking change|deprecat)', re.IGNORECASE)` for body keywords
   - Weights as class constants: `WEIGHTS = {"sender": 40, "subject_mc": 30, "body_keywords": 30}`
   - Threshold as class constant: `THRESHOLD = 70`

   - `classify(self, email: Email) -> ClassificationResult` method:
     - Check each signal against compiled patterns
     - Accumulate weighted score
     - Record matched signal names
     - Log classification decision: `logger.info(f"Classified email {email.id[:20]}...: score={score}, signals={signals}, is_major={is_major}")`
     - Return ClassificationResult

   - `classify_batch(self, emails: list[Email]) -> tuple[list[Email], list[Email]]` method:
     - Iterate emails, classify each
     - Split into (regular_emails, major_update_emails)
     - Log summary: `logger.info(f"Classification complete: {len(regular)} regular, {len(major_updates)} major updates")`
     - Return tuple

Follow existing codebase conventions:
- Module-level logger: `logger = logging.getLogger(__name__)`
- Double quotes for strings
- Type hints on all parameters and returns
- Docstrings in Google-style format (Args/Returns)
- Import from `.ews_client import Email`
- Private methods with single underscore prefix

Do NOT:
- Add LLM classification (that is Phase 4 scope)
- Import or reference Config (classifier should be config-free, patterns are class constants)
- Add any extraction methods for message_id, service, impact_level, deadline (those are Phase 3 scope)
- Make this overly complex - keep it focused on detection only
