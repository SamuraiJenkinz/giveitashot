# Milestone v1.0: Major Updates Digest

**Status:** SHIPPED 2026-02-26
**Phases:** 1-5
**Total Plans:** 11

## Overview

This milestone extends InboxIQ with a dual-digest system: detect M365 Message Center major update emails, exclude them from the regular digest, and deliver a separate admin-focused digest with deadline emphasis, affected services, and AI-extracted action items. The architecture follows classification-first design, with phases ordered by dependency: detection foundation, state/config infrastructure, digest content formatting, AI enhancement, integration testing.

## Phases

### Phase 1: Detection Foundation

**Goal**: Reliably identify M365 Message Center major update emails and exclude them from regular digest
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01-PLAN.md -- EmailClassifier module with multi-signal weighted detection and unit tests
- [x] 01-02-PLAN.md -- Pipeline integration: wire classifier into main.py, exclude major updates from regular digest

**Details:**
- Multi-signal weighted scoring: sender pattern, subject MC# regex, body keyword detection
- Classification threshold 70% requires 2+ strong signals to reduce false positives
- MC pattern accepts 5-7 digit numbers for format flexibility
- Classification failure fallback treats all emails as regular (non-blocking)
- 26 tests passing at phase completion

### Phase 2: Configuration and State Management

**Goal**: Infrastructure supports dual-digest workflows with separate recipients and independent state tracking
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:
- [x] 02-01-PLAN.md -- Major update recipient config, feature toggle, validation, and .env.example
- [x] 02-02-PLAN.md -- Digest-type-aware state management, migration, and CLI flags

**Details:**
- Presence-based feature toggle: MAJOR_UPDATE_TO non-empty activates major digest
- No fallback from MAJOR_UPDATE_TO to SUMMARY_TO (different audiences)
- Per-digest-type state tracking with automatic migration from old format
- Old last_run key preserved after migration for rollback safety
- Corruption isolation: one digest type cannot affect the other
- CLI flags: --regular-only and --major-only for selective execution
- 48 tests passing at phase completion

### Phase 3: Digest Content Extraction

**Goal**: Major updates digest displays all essential Message Center information in professional HTML format
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:
- [x] 03-01-PLAN.md -- MessageCenterExtractor: field extraction, urgency calculation, deduplication, and tests
- [x] 03-02-PLAN.md -- HTML digest formatter, pipeline integration, and major digest send flow

**Details:**
- MC ID extraction, multi-format date parsing (MM/DD/YYYY and "Month DD, YYYY")
- Urgency tiers: <=7 days Critical, <=30 days High, >30 days Normal
- Deduplication by MC ID keeping latest by received_datetime
- Service/category normalization (title case services, uppercase categories)
- HTML gradient header, urgency-grouped sections, left-border traffic light colors
- Error isolation: major digest failure does not crash or prevent regular digest
- 107 tests passing at phase completion

### Phase 4: AI Enhancement

**Goal**: AI extracts actionable admin tasks and displays time-sensitive deadline countdowns
**Depends on**: Phase 3
**Plans**: 2 plans

Plans:
- [x] 04-01-PLAN.md -- ActionExtractor module with Pydantic models, Azure OpenAI structured outputs, and unit tests
- [x] 04-02-PLAN.md -- Wire action items into major digest HTML and pipeline integration

**Details:**

## Milestone Summary

**Decimal Phases:** None (no urgent insertions required during this milestone)

**Key Decisions:**

- Separate email for major updates (different audience, different emphasis)
- Different recipients for major updates (admin team vs general team)
- Same hourly schedule (one scheduled task handles everything)
- Email-based detection, not Graph API (avoid new API dependency)
- Classification threshold 70% (2+ strong signals)
- Presence-based feature toggle (no separate ENABLE flag)
- Graceful degradation throughout (classification, AI extraction, digest pipeline)
- Error isolation (major digest failure never blocks regular digest)

**Issues Resolved:**

- MC email sender pattern expanded for production format (o365mc@microsoft.com and *@email2.microsoft.com)
- Keyword detection extended to subject+body (real MC emails have keywords in subject line)
- Multi-format date parsing for varied Message Center date formats

**Issues Deferred:**

- EWS to Graph API migration (EWS deprecated Aug 2026) — deferred to v2.0
- Digest enhancements (impact summaries, service grouping, retirement timelines) — deferred to v1.1+

**Technical Debt Incurred:**

- Missing Phase 4 VERIFICATION.md formal verification artifact (work verified via summaries and Phase 5 integration tests)
- Config.validate() does not validate MAJOR_UPDATE_TO format (intentional — matches SUMMARY_TO behavior)
