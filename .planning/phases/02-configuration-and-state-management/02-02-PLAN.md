Refactor StateManager in src/state.py to support per-digest-type state tracking while maintaining backwards compatibility.

**StateManager changes (src/state.py):**

1. Modify `get_last_run()` to accept `digest_type: str = "regular"` parameter:
   - Build key as `f"{digest_type}_last_run"` (e.g., "regular_last_run", "major_last_run")
   - Look up key in self._state
   - If found, parse with datetime.fromisoformat() and return
   - If not found or parse fails, return None (treat as first run for that digest type)
   - Wrap in try/except for ValueError/TypeError — log warning and return None on corruption

2. Modify `set_last_run()` to accept `digest_type: str = "regular"` parameter:
   - Build key as `f"{digest_type}_last_run"`
   - Store timestamp under that key
   - Log at info level: f"Updated {digest_type} last run time: {timestamp.isoformat()}"

3. Add backwards-compatible state migration in `_load()`:
   - After loading JSON, check if "last_run" key exists AND "regular_last_run" does NOT exist
   - If so, copy value: `self._state["regular_last_run"] = self._state["last_run"]`
   - Keep old "last_run" key for rollback safety (do NOT delete it)
   - Log at info level: "Migrated state: last_run -> regular_last_run"
   - Call self._save() to persist migration

4. Modify `clear()` to accept optional `digest_type: str | None = None` parameter:
   - If digest_type is None (default), clear ALL state (existing behavior)
   - If digest_type is provided, only remove that specific key (e.g., "regular_last_run")
   - This allows clearing one digest's state without affecting the other

**main.py changes:**

1. Add two new CLI arguments after the existing --clear-state argument:
   ```python
   parser.add_argument(
       "--regular-only",
       action="store_true",
       help="Send only regular digest (skip major update digest)"

   parser.add_argument(
       "--major-only",
       action="store_true",
       help="Send only major update digest (skip regular digest)"
   )
   ```

2. Update the fetch mode section to use digest_type when calling state:
   - Change `since = state.get_last_run()` to `since = state.get_last_run(digest_type="regular")`
   - This is a backwards-compatible change — regular digest behavior is unchanged

3. Update the state update after successful send:
   - Change `state.set_last_run()` to `state.set_last_run(digest_type="regular")`

4. Do NOT implement the full major digest send logic yet (that's Phase 3). Just add the CLI flags and wire up the state calls for regular digest. The major digest send path will be added when the content formatting is ready.

