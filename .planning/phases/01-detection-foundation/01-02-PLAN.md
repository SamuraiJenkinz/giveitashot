- Add import at top: `from typing import Optional, Any`
- Add field to Email dataclass AFTER `has_attachments`:
  ```python

  ```

  ```python

      return self.classification.is_major_update
  ```

**2. Modify `src/main.py` - Insert classification between fetch and summarize:**

Add the classification step to the main() function. The key change: after fetching emails and before summarizing, classify them and split into regular vs major update streams.

- Add import: `from .classifier import EmailClassifier`

- After the line `emails = ews_client.get_shared_mailbox_emails(Config.SHARED_MAILBOX, since=since)` and the existing empty-check, add classification:

  ```python
  # Classify emails to detect Message Center major updates
  logger.info("Classifying emails...")
  classifier = EmailClassifier()
  try:
      regular_emails, major_update_emails = classifier.classify_batch(emails)
      logger.info(f"Classification: {len(regular_emails)} regular, {len(major_update_emails)} major updates")
  except Exception as e:
      logger.warning(f"Classification failed, treating all as regular: {e}")
      regular_emails = emails
      major_update_emails = []
  ```

- Log major update details if any found:
  ```python
  if major_update_emails:
      logger.info("Major updates detected (will be processed in future digest):")
      for mu_email in major_update_emails:
          logger.info(f"  - {mu_email.subject}")
  ```

- Change the existing summarization call to use `regular_emails` instead of `emails`:
  - Replace: `summary = summarizer.summarize_emails(emails)`
  - With: `summary = summarizer.summarize_emails(regular_emails)`

- Update the "no emails" check after classification. If regular_emails is empty but major_update_emails has items, log that regular digest is skipped but major updates were detected:
  ```python
  if not regular_emails:
      if major_update_emails:
          logger.info(f"No regular emails to summarize ({len(major_update_emails)} major update(s) detected, digest pending Phase 2)")
      else:
          logger.info("No new emails found - skipping summary")
      # ... existing early return logic
  ```

- Update the email count log to show regular vs total:
  ```python

  ```

- In the dry-run section, also log major updates:

  ```python
      for mu_email in major_update_emails:
          logger.info(f"  - {mu_email.subject}")
  ```

- Wrap the entire classification in try/except so classification failures NEVER prevent the regular digest from sending. If classification fails, fall back to treating all emails as regular (existing behavior preserved).
