**Method 1: format_major_updates_html(self, updates: list[MajorUpdateFields]) -> str**

Generates a complete HTML email for the major updates digest. Reuse the existing color palette dict from format_summary_html (extract it into a class-level method `_get_color_palette()` that both methods call, OR duplicate the colors dict inline — prefer the helper method to stay DRY).

HTML structure (all table-based layout, all inline CSS, 600px width):

1. **Outer wrapper** — Same as regular digest: centered table, bg_light background, 600px inner table with border-radius and box-shadow.

2. **Header** — Gradient background like regular digest but use danger color (#ea4335) to primary (#1a73e8) gradient to visually distinguish from regular digest. Title: "Major Updates Digest" with shield/warning emoji. Subtitle: today's date formatted as "Monday, February 24, 2026".

3. **Stats bar** — Left side: "Major Updates" label with urgency breakdown text like "2 Critical, 3 High, 5 Normal". Right side: total count in large font. Use same layout as regular digest stats bar. Count each urgency tier from the updates list.

4. **Urgency tier sections** — For each tier (Critical, High, Normal) that has updates, render a section:
   - Section header: tier name with colored dot indicator (using urgency color). E.g., "CRITICAL — Action Required Within 7 Days". Skip section if tier has 0 updates.
   - Critical header text: "CRITICAL - Immediate Action Required"
   - High header text: "HIGH PRIORITY - Action Required Within 30 Days"
   - Normal header text: "NORMAL - Informational Updates"

5. **Update cards** — For each MajorUpdateFields in the tier (sorted by action date, soonest first; None dates sort last):
   - Left border accent: 4px solid border in tier color (Critical=#ea4335, High=#fbbc04, Normal=#34a853)
   - Background: bg_light (#f8f9fa)
   - MC ID badge: Bold, primary color (#1a73e8). If is_updated=True, show small "UPDATED" badge next to MC ID in amber/warning color.
   - Action date: If present, show "Action Required: March 15, 2026" formatted with strftime("%B %d, %Y"). If date is past, show "(OVERDUE)" in red. If None, show "No deadline specified" in text_light color.
   - Days remaining: If action date present, show "(X days remaining)" or "(X days overdue)" calculated from today.
   - Affected services: Show as inline pill badges (rounded background spans) using bg_light with border. If empty, show "Services not specified" in text_light.
   - Categories: Show as inline pill badges in uppercase. Use urgency tier color background with white text for the primary category, muted for others. If empty, omit.
   - Published date: "Published: Feb 24, 2026" using received_datetime formatted short.
   - Last updated: Only show "Updated: Feb 25, 2026" if is_updated=True.
   - Body preview: Show the body_preview text (already truncated to 200 chars by extractor) in text_medium color, 13px font.

6. **Footer** — Same pattern as regular digest: bg_light background, centered text, "Generated automatically by InboxIQ Major Updates Digest".

Date formatting for display:
- Action date: strftime("%B %d, %Y") — "March 15, 2026"
- Published/Updated: strftime("%b %d, %Y") — "Feb 24, 2026"
- Days remaining: (action_date.date() - datetime.now().date()).days

Handle the zero-updates case: If updates list is empty, return empty string (caller should not send). Do NOT generate an "all clear" template for major digest.
