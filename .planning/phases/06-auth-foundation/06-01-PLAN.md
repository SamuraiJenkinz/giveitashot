<task type="auto">
  <name>Task 2: Rewrite auth.py as GraphAuthenticator and add unit tests</name>
  <files>src/auth.py, tests/test_auth.py</files>
  <action>
Rewrite src/auth.py:

1. Update module docstring to: "OAuth 2.0 Authentication module for Microsoft Graph API using client credentials flow."

2. Remove the `from exchangelib import OAuth2Credentials, Identity` import entirely. Keep `import msal` and `from .config import Config`.

3. Add module-level scope constant: `GRAPH_SCOPE = ["https://graph.microsoft.com/.default"]`

4. Keep `AuthenticationError` exception class unchanged.

5. Create `GraphAuthenticator` class (replaces EWSAuthenticator):
   - Same `__init__` as before: calls `Config.validate()`, initializes `self._app = None`. Remove `self._access_token` (unnecessary state).
   - Same `app` property (lazy init of ConfidentialClientApplication using Config.CLIENT_ID, Config.CLIENT_SECRET, Config.get_authority()).
   - `get_access_token()` method: Use ONLY `self.app.acquire_token_for_client(scopes=GRAPH_SCOPE)` — do NOT call `acquire_token_silent` first (redundant since MSAL 1.23, see RESEARCH.md). Return `result["access_token"]` as plain string on success. Raise `AuthenticationError` with error/error_description on failure.
   - `clear_cache()` method: Set `self._app = None`, log message.
   - Do NOT include `get_ews_credentials()` — that method is EWS-specific and the Graph authenticator returns plain tokens.

6. Add backward-compat alias at module level after the class: `EWSAuthenticator = GraphAuthenticator  # Alias — remove when main.py is updated (Phase 8)`. This ensures `from .auth import EWSAuthenticator` in main.py continues to work.

7. Add a comment noting client secret rotation: `# NOTE: Client secret expires per Azure AD app registration (typically 24 months). Monitor expiry.`

Create tests/test_auth.py (new file):

1. Import GraphAuthenticator, AuthenticationError, GRAPH_SCOPE, and EWSAuthenticator (alias).

2. Use `unittest.mock.patch` to mock `msal.ConfidentialClientApplication` and `Config.validate`.

3. Tests to write:
   - `test_graph_scope_is_correct`: Assert `GRAPH_SCOPE == ["https://graph.microsoft.com/.default"]`.
   - `test_get_access_token_returns_string`: Mock `acquire_token_for_client` to return `{"access_token": "test-token-123"}`. Assert return value is `"test-token-123"` and is a plain `str`.
   - `test_get_access_token_raises_on_error`: Mock `acquire_token_for_client` to return `{"error": "invalid_client", "error_description": "Bad secret"}`. Assert `AuthenticationError` is raised with "invalid_client" in the message.
   - `test_get_access_token_raises_on_exception`: Mock `acquire_token_for_client` to raise `Exception("network error")`. Assert `AuthenticationError` is raised.
   - `test_get_access_token_uses_graph_scope`: Mock `acquire_token_for_client` to return success. Assert it was called with `scopes=GRAPH_SCOPE` (i.e., the Graph scope, not the EWS scope).
   - `test_acquire_token_silent_not_called`: Mock the MSAL app. Call `get_access_token()`. Assert `acquire_token_silent` was NOT called (verifying we removed the redundant call).
   - `test_ews_authenticator_alias`: Assert `EWSAuthenticator is GraphAuthenticator` (backward compat alias works).
   - `test_clear_cache_resets_app`: Create authenticator, access `.app` to initialize it, call `clear_cache()`, assert `_app` is None.

4. Use a pytest fixture that patches Config.validate to be a no-op and patches Config attributes (TENANT_ID, CLIENT_ID, CLIENT_SECRET) with test values.
